CI/CD Pipeline + Observability Stack
Designed a fully automated CI/CD pipeline with integrated observability, ensuring every deployment is tested, validated, and monitored before reaching production..
⚙ CI/CD
Jenkins: 8-stage pipeline. Checkout, unit tests, multi-stage Docker build, SHA tag, push ECR, deploy, health check, verify. No manual steps.
🌐 Traffic
Nginx > FastAPI: Nginx is the only exposed port. FastAPI runs isolated on the internal network. Nginx waits for FastAPI healthy before accepting traffic.

Prometheus + Node Exporter: scrapes FastAPI /metrics every 15s. Grafana shows request rate, error rate, CPU, memory live.
🔔 Alerting
Alertmanager > Slack: fires on uptime and error rate. Handles deduplication, grouping, and routing.

Phase 2Complete
Intelligent Alert Automation
Built an automated incident response system where alerts trigger remediation, recovery validation, and escalation without manual intervention..
🔍 Blackbox
Blackbox Exporter: probes HTTP endpoints externally every 15s. N8NDown alert fires if the automation layer itself goes down.
🤖 n8n
Alert enrichment + remediation: parses payload, adds Grafana URL and runbook, restarts containers via Docker API, verifies recovery via Prometheus, posts timeline to Slack.
🔒 Proxy
Docker Socket Proxy: n8n calls Docker API through a proxy exposing only container restart. Least privilege applied to the automation layer.
📡 Fallback
Dual delivery: critical alerts route to n8n AND Slack simultaneously. If n8n is down, Alertmanager delivers directly. Critical alerts are never silently dropped.

Phase 3Complete
AWS Cloud Infrastructure with Terraform
Implemented a fully automated cloud deployment pipeline on AWS, where a single git push triggers build, versioning, and deployment to ECS Fargate through Terraform-managed infrastructure..
☁ Network
VPC 10.0.0.0/16 across two AZs in eu-central-1. Public and private subnets, Internet Gateway, Security Groups with least privilege.
💻 ECS
ECS Fargate: no servers to manage. ECS replaces failed tasks automatically. ALB health checks every 30s, removes unhealthy tasks from rotation.
🌐 ALB
Application Load Balancer: permanent DNS, zero-downtime rolling deployments. Auto Scaling adds tasks at CPU >70%.
🔒 IAM
Least privilege throughout: EC2 gets ECR ReadOnly. Jenkins scoped to ECR push and specific ECS service ARN. ECS execution role limited to ECR pull and CloudWatch logs.
📋 IaC
Terraform: all infrastructure as code. Remote state in S3 (versioned, AES256) with DynamoDB locking. S3 and DynamoDB bootstrapped manually first.

// Failures & Learnings
What broke. What I learned.
CI/CD
Jenkins deployed stale image silently
fix
Workspace was cached. Added cleanup at pipeline start and forced fresh checkout on every build.
Networking
Healthy internally, unreachable externally
fix
ALB target pointed to wrong port. Separated internal and external verification steps.
Registry
GHCR to ECR migration broke the pipeline
fix
ECR uses aws ecr get-login-password. GHCR uses a static PAT. Completely different auth. Refactored the entire push stage.
Terraform
Remote state bootstrap chicken-and-egg
fix
Backend needs S3 and DynamoDB before init. Created both manually via AWS CLI first, then configured backend.tf.
ECS Deploy
ECS kept deploying the old image
fix
Called update-service before register-task-definition. Fixed the order, passed the new task ARN explicitly.
IAM
ECS could not pull image, silent 403
fix
Execution role had the right permission policy but trust policy was missing ecs-tasks.amazonaws.com. Fixed the trust relationship.
